# Mockup catalog

**Status:** ACCEPTED (living index — update it in the same change whenever a mockup is added, or a listed one is superseded/ships)

Interactive HTML mockups live in `mockups/` at the repo root. This is the map of what each one is for, what it promotes/supersedes, and its status relative to the shipped app — read the entry for the surface you're touching, not the whole file. Moved out of the root `CLAUDE.md` (ADR-0097) so a task that never touches design/mockups doesn't load this list.

On conflict between an older mockup and `design-language.md` or an accepted ADR, the newer accepted doc wins — each entry below says so where it applies.

- [`mockups/trip-dashboard-v2.html`](../../mockups/trip-dashboard-v2.html) — the interactive **Trip-mode** design reference _(predates ADR-0028; colors retrofitted — on conflict the design docs win)_. Its HOME section is superseded by `trip-home-v3.html`, its INDEX section by `trip-index-v1.html`, and its DAY section by `trip-mode-day-view-v1.html`; it remains the reference for the one tab that hasn't gotten a dedicated mockup — map.
- [`mockups/trip-mode-day-view-v1.html`](../../mockups/trip-mode-day-view-v1.html) — the **Day-view tab** design exploration for ADR-0043 (now-line + derived phases, phase-scoped verbs, archive chrome): two review rounds recorded in this file; the shipped now-line is a quiet hairline, superseding the mockup's louder first-pass (dark chip + glow + pulse).
- [`mockups/logo-v1.html`](../../mockups/logo-v1.html) — historical design record for the **app logo** (ADR-0087, Accepted): four explored directions (waypoint marker with an amber core / marker holding a mini departure board / brighter board-tile evolution / RTL route to an amber destination), the chosen one fitted into the landing + invite screens. Kept for the design rationale, not a build spec.
- [`mockups/trip-index-v1.html`](../../mockups/trip-index-v1.html) — the **Index tab** (bookings + documents), promoting ADR-0047 (linkage, delete/unlink, per-file docs, notes + hotel WiFi), ADR-0048 (merged edit sheet with the shared TimePicker/IconPicker; transport origin/destination place fields), and ADR-0049 (mode = chrome only; a during-trip past/upcoming split; a read-only washed archive). Interactive: the demo toggle switches during / empty / archive; tapping a booking opens the merged edit sheet, its delete opens the delete/unlink prompt. The booking-entry _form_ lives in `plan-mode-v1.html`. Post-build fixes to this screen are mocked in `index-fixes-v1.html` below.
- [`mockups/index-fixes-v1.html`](../../mockups/index-fixes-v1.html) — **post-build fixes** to the shipped Index (2026-07-17 triage), promoting three _Proposed_ ADRs: ADR-0052 (documents: mobile-first PDF open/download, per-row ⋯ → rename/change-type/replace/delete, upload+load spinner, cause-aware errors, four distinct type badges), ADR-0053 (bookings: a read-only detail view + ⋯ menu → edit/delete like the event card, and the merged edit surface reachable from the linked event), ADR-0054 (a hotel / multi-day booking is an ambient span shown as backdrop, excluded from the day-at-a-glance rail + count). Demo toggle: מסמכים / הזמנה / לוז. Supersedes `trip-index-v1.html` only on booking tap-to-detail (the rest of that file stands).
- [`mockups/index-findability-v1.html`](../../mockups/index-findability-v1.html) — exploration for **finding a booking on a busy Index** (no ADR yet — direction under discussion): a category filter chip row over the bookings list (neutral, not category-tinted — the color budget stays reserved), a search box (icon → text input) over title/confirmation code, past bookings collapsed by default mirroring ADR-0061's Plan Home "completed checks" toggle exactly, and the bookings/documents sections each independently shrinkable via one big tappable title (title + count only when collapsed), replacing an earlier sticky jump-nav draft. The "⬇ עובד אופליין" badge moved off the bookings title into its own page-level line above both sections. Scoped to bookings only; documents keep their existing type-grouping. Demo toggle: טיול עמוס (17 bookings, the target case) vs טיול קטן (chips/search step back when there's little to filter). Directional glyphs (the section chevron, the route arrow, the trip-switch caret) are the real `ui/NavArrow.tsx`/`ui/Icon.tsx` SVGs, not Unicode arrow characters — a `‹`/`›`/`→` glyph is `Bidi_Mirrored` and silently flips direction inside `dir="rtl"` (confirmed by rendering test), which is exactly the bug those two components exist to avoid.
- [`mockups/index-findability-split-v1.html`](../../mockups/index-findability-split-v1.html) — a **side-by-side comparison companion** to the file above, exploring the "separate screens navigable from the Index" alternative Assaf floated: an Index landing with two peer tiles (🎫 הזמנות / 📎 מסמכים, each showing its count + a one-line preview), tapping one pushes a dedicated full screen with an icon-only back button + "אינדקס" label (matching the shipped `.back` pattern in TripSettings.tsx/AllTrips.tsx). Same data + same chips/search/past-collapse as v1, but the per-section shrink chevron is gone — there's no second section left to compete for space once each list owns a whole screen. Also carries a **scale check**: three disabled "בקרוב" placeholder tiles (📝 הערות / 🔎 מחקר / 🖼️ מדיה) added to see how a 5-tile landing reads once the Index grows past bookings+documents — Assaf wants to add notes/research/media later, and that growth trajectory is the main argument for the split (a landing scales by adding a tile; a single page scales by stacking another whole section+chips+search+add-button, which crowds past ~2 types). Flags one open naming question: "מחקר" already names the shipped Map-tab research feature (place search + pin → maybe-shelf); an Index research tile needs to mean something else or it'd duplicate that surface. Not a decision yet — a comparison point that also trades the combined page's length for a navigation layer (extra tap, a new back-target) and would need to revisit ADR-0049 §3's "bookings and documents are peers on one screen." Same real-SVG-not-glyph fix as v1 applies to its tile chevrons, back arrow, route arrow, and trip-switch caret.
- [`mockups/trip-home-v3.html`](../../mockups/trip-home-v3.html) — the redesigned **Trip-mode Home** (ADR-0045): board hero unchanged, quick-access reworked to three real shortcuts (next code / WiFi / documents; navigate-to-next deferred to the maps work), and the fixture weather/FX/budget glance row replaced by a derived "day at a glance" card. Real-data-only home. The quick-access tiles deep-link into the Index and follow the derived-vs-managed empty-state rule (ADR-0050): derived tiles (next code, WiFi-from-hotel-Booking) vanish when there's no source; the managed documents tile stays with a ＋ invite. Companions: `mockups/trip-home-glance-options-v1.html` (glance counting-model options — lead-with-"נותרו" chosen) and `mockups/trip-home-refinements-v1.html` (quick-access derived-vs-managed states incl. the no-hotel WiFi reflow per ADR-0050, the glance under overlapping/contained events → counts top-level blocks, and the empty-day teach state).
- [`mockups/booking-presentation-v1.html`](../../mockups/booking-presentation-v1.html) — historical design record for **booking presentation across the Trip-mode Home & Index** (ADR-0059 applying ADR-0063's per-category time-behaviour profile — **Accepted, shipped** session 33): the board hero's transition-moment surfacing (hotel check-in/out, transport departure/arrival), the "inside a booking" ambient/teal treatment, the shared row/detail/hero grammar, and the glance's uncounted check-in/out markers (ADR-0054 rebased). Kept for the design rationale, not a build spec.
- [`mockups/glance-transition-labels-v1.html`](../../mockups/glance-transition-labels-v1.html) — first exploration of the **glance transition-label** clutter (the amber-pill-per-edge markers laddering on a busy day): current-vs-three-alternatives (journey brackets / minimal pins / legs line). - [`mockups/glance-timeline-labels-v2.html`](../../mockups/glance-timeline-labels-v2.html) — the **unified glance rail-annotation grammar** (ADR-0077): one pill primitive where tint = family (amber time-anchor / neutral structure) and the connector = shape (bar+feet span / stem point / tick block-note); time-anchors above the rail, structure below; a same-day flight/ferry is a span, a lone hotel check-in a point, overlap/containment block-notes; a legs-line collapse past two lanes, plus edge anchor-in and short-span handling. Includes a three-grammar head-to-head (two-bands chosen) and scenario stress-tests (ordinary / arrival edges / crowded departure / short span).
- [`mockups/when-field-v1.html`](../../mockups/when-field-v1.html) — the **WhenField date/time standard** (ADR-0083): the one date/time entry primitive every form uses. Before/after (the cramped native `datetime` box that clipped the date + showed AM/PM → the standard), the `span` variant's two [native full-width date + compact tap-to-open amber time] endpoints with an auto-closing time panel, a `+N` days badge, and a duration read-out. The `day` variant (native date + amber TimePicker) is the event grammar unchanged.
