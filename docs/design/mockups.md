# Mockup catalog

**Status:** ACCEPTED (living index — update it in the same change whenever a mockup is added, or a listed one is superseded/ships)

Interactive HTML mockups live in `mockups/` at the repo root. This is the map of what each one is for, what it promotes/supersedes, and its status relative to the shipped app — read the entry for the surface you're touching, not the whole file. Moved out of the root `CLAUDE.md` (ADR-0097) so a task that never touches design/mockups doesn't load this list.

On conflict between an older mockup and `design-language.md` or an accepted ADR, the newer accepted doc wins — each entry below says so where it applies.

- [`mockups/trip-dashboard-v2.html`](../../mockups/trip-dashboard-v2.html) — the interactive **Trip-mode** design reference _(predates ADR-0028; colors retrofitted — on conflict the design docs win)_. Its HOME section is superseded by `trip-home-v3.html`, its INDEX section by `trip-index-v1.html`, its DAY section by `trip-mode-day-view-v1.html`, and its MAP section by `map-tab-v1.html` below (ADR-0108) — so all four tabs now have a dedicated mockup and this file is kept only for its cross-tab reference value.

- [`mockups/index-findability-v1.html`](../../mockups/index-findability-v1.html) — historical design record for the **single-scrolling-page findability direction**, superseded by `index-findability-split-v1.html` below (ADR-0098, Accepted — not yet built): a category filter chip row over the bookings list (neutral, not category-tinted — the color budget stays reserved), a search box (icon → text input) over title/confirmation code, past bookings collapsed by default mirroring ADR-0061's Plan Home "completed checks" toggle exactly, and the bookings/documents sections each independently shrinkable via one big tappable title. Directional glyphs are the real `ui/NavArrow.tsx`/`ui/Icon.tsx` SVGs, not Unicode arrow characters — a `‹`/`›`/`→` glyph is `Bidi_Mirrored` and silently flips direction inside `dir="rtl"` (confirmed by rendering test). Carries the motion pass described below.
- [`mockups/index-findability-split-v1.html`](../../mockups/index-findability-split-v1.html) — the **accepted Index redesign** (ADR-0098, Accepted, shipped session 60): the Index becomes a landing with two peer tiles (🎫 הזמנות / 📎 מסמכים, each showing its count + a one-line preview), tapping one pushes a dedicated full screen with an icon-only back button + "אינדקס" label (matching the shipped `.back` pattern in TripSettings.tsx/AllTrips.tsx). Real `NavArrow`/`Icon` SVGs throughout (see ADR-0098's arrow-mirroring note), plus a **motion pass**: the past-list animates open/shut instead of snapping via `display:none`, a chip/search change reveals matching rows with a small per-row stagger, and every tappable control gets a brief scale-down on tap — motion instead of more color/content, since the color budget and Home's board-hero stay off-limits (ADR-0028); all of it respects `prefers-reduced-motion`. Also carries a **scale check** (not a shipped feature): three disabled "בקרוב" placeholder tiles (📝 הערות / 🔎 מחקר / 🖼️ מדיה) testing how a 5-tile landing reads, since Assaf may want more content types later — nothing beyond bookings/documents is decided or in scope; ADR-0098 explicitly flags that "מחקר" already names the shipped Map-tab research feature and would need a distinct meaning before it's ever real. Demo toggle: טיול עמוס (17 bookings) vs טיול קטן (a handful). Post-ship polish feedback on this build is explored in `index-bookings-compact-v2.html` below.
- [`mockups/index-bookings-compact-v2.html`](../../mockups/index-bookings-compact-v2.html) — the **accepted bookings header/search redesign** (ADR-0100, Accepted, shipped session 62), reacting to polish feedback on the shipped ADR-0098 build ("the search bar and the closeby area [don't] read as polished"): the back-row and title/count row MERGE into one compact line (`[← beside "אינדקס"] ... [count]`, arrow pointing right — fixing a wrong-direction/orphaned-arrow arrangement on Assaf's own reference screenshot); the category chips + search become one denser row (each chip carries label+icon+count, e.g. "רכבת 🚄 2"; a minimal real-SVG, non-circular search icon sits fixed at the row's end and, when opened, covers the chip strip in place with a small animation rather than opening a second row); the chip row's edges fade (`scroll-snap` + a `mask-image` gradient) instead of hard-clipping a partially-visible chip; and a **Plan vs. Trip mode color palette** — the Index's own accent (selected chip / search icon, not per-category tinting) follows the active mode: neutral ink in Trip (unchanged), plan violet in Plan, extending ADR-0028's existing per-mode selected-accent rule (the day-strip's own selection already does this) rather than adding new color policy. Superseded, for this header/chip/search area only, the interim search-bar-replaces-title-row shape shipped straight to `IndexBookingsView.tsx` in the ADR-0098 follow-up session — ported into `ui/IndexBookingsView.tsx`/`ui/IndexBackRow.tsx`/`ui/IndexDocumentsView.tsx`/`screens.css` in session 62 (`docs/planning/2026-07-21-session-62-index-bookings-header-redesign-build.md`).

- [`mockups/loading-states-v1.html`](../../mockups/loading-states-v1.html) — the **loading states** design, promoting [ADR-0105](../decisions/0105-loading-states-design.md) (**Accepted, boot + snapshot shipped session 69**): the three loading moments as one language, extending the U-10 feedback family (ADR-0078). **Boot** (`shell.booting`) is now `ui/feedback/BootScreen` — the loud "board power-on": the dormant departure board warming up (the glow _ramps_ via `.fb-boot-mark::before`, never the reserved live pulse; mono carries the numerals since JetBrains Mono has no Hebrew glyphs). Amber is kept a **time accent** throughout — a full amber ground was tried and rejected as off-budget (design-language / ADR-0028). Grounds in ADR-0028/0052/0078/0092 and ADR-0016/0040 (mode is derived, so boot can't know it). Build session note: `docs/planning/2026-07-23-session-69-loading-states-build.md`.
