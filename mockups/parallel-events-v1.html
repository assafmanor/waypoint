אירועים במקביל · עיצוב מאושר

חפיפת זמנים היא מצב לגיטימי (ADR-0011). מודל אחיד: אם אירוע אחד מכיל את כל השאר → קינון (זהה בטיול ובתכנון, בדרך כלל מכוון). אחרת → אשכול: בטיול סוגר דק שקט, בתכנון כרוך בסגול עם "הזז" — ובמכוון שונה מפער (פער = מקום ריק להוסיף; חפיפה = כרוך באותו זמן). הלוח שומר על גיבור אחד + "ועוד N עכשיו".

הכלה → קינון · זהה בטיול ובתכנון (בדרך כלל מכוון, בלי "הזז")

כללי העיצוב (למימוש)

  • קבוצה = רצף אירועים שחופפים בזמן (interval-merge; רכיב קשירות). אירוע בודד ללא חפיפה נשאר שורה רגילה. end === start (גב-אל-גב) אינו חפיפה.
  • מבנה רקורסיבי (יער־הכלה). הורה = הטווח הקטן ביותר שמכיל את האירוע. בין אחים באותה רמה: מי שחופף חלקית מקובץ לאשכול. כך קינון ואשכול מתחברים — קינון יכול להכיל אשכול (חפיפה בתוך הכלה) ואיבר באשכול יכול להיות קינון (הכלה בתוך חפיפה).
  • עומק תצוגה מוגבל. הזחה עד 2 רמות; רמות עמוקות יותר משוטחות פנימה ("כולל N", ניתן־להרחבה). טווחים שווים = עמיתים באשכול, לא קינון. המודל מדויק, התצוגה חסומה לרוחב טלפון.
  • קינון — זהה בשני המצבים. מעטפת + תוכן מקונן ("כולל N"), ניטרלי, בלי "הזז" (בדרך כלל מכוון). מבנה ודיו בלבד; אמבר נשאר לזמן/עכשיו.
  • אשכול — טיול שקט, תכנון מסמן. טיול = סוגר דק ניטרלי + "בו-זמנית". תכנון = כרוך בסגול + "הזז", ובמכוון שונה מפער (אוורירי, מוסיף, בין־שורות).
  • "הזז" = ריפל הפוך. מזיז אירוע גמיש בלבד (קשיח = עוגן), משך נשמר. כשיש כמה גמישים — בוחרים מה להזיז (קשיח מופיע כעוגן מושבת); כניסה מתג־החפיפה מדלגת ישר לאירוע שנבחר. ואז משבצת נקייה לפני/אחרי העוגן בהקשה אחת, או קובע־זמן מדויק (ADR-0036); אופטימי + ביטול. חפיפה חדשה במורד היום מפעילה את הריפל הקיים. שתי־קשיחות שחופפות = אזהרת ⚠️ בלבד (אין "הזז").
  • דקדוק קשיח/רך נשמר בתוך האשכול (קשיח = כרטיס מלא + 🔒 + קוד; רך = מקווקו).
  • לוח = גיבור אחד. "עכשיו" ראשי לפי: קשיח > מסתיים־ראשון > מתחיל־ראשון; השאר ב"ועוד N". פיצול־קבוצה (בלי גיבור) הוא וריאנט.
  • מודל נתונים: deriveNow יחזיר nowAll[] / nextAll[] (קבוצות בו-זמניות), נגזר מהשעון, לא נשמר (ADR-0018); עוזר משותף clusterByTime מזין גם את היום וגם את הלוח.
