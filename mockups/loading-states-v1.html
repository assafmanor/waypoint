Three loading states, one loading language

The app has three loading moments and they read as three unrelated things today: a bare word, a generic shimmer, and an inline row status. This lays out one motif in three weights — a filling departure-line that echoes the day-progress and readiness bars already in the product — in the real tokens. Boot is the loud board power-on (light + a designed-not-built dark); the snapshot pre-draws the real Home per mode; the upload row stays inline sync grammar.

Amber is an accent, not a ground

Amber = time, used sparingly. Here it’s only the clock and a small halo — the ground is the real --screen neutral the whole app sits on. That’s what a full amber field broke.

Light trades loud for genuinely light

The board’s loudness comes from a dark ground under an amber glow — kept in the dark theme. Light theme deliberately goes the other way: cool paper reads calm and unmistakably day, so the two themes feel like different times of day, not near-identical.

Theme is knowable at boot; mode is not

Light/dark is a device setting available immediately, so theming boot is honest. Trip/Plan mode is derived from a trip the snapshot hasn’t loaded yet — so boot reads as brand, not mode, and gets no mode variant.

Warm-up is not the live pulse

Pulse means “live this minute” and is reserved. The glow ramps as a warm-up, distinct from the sharp blip, so a boot never claims to be live. Mono carries the numerals (no Hebrew glyphs), so the word stays in the heading face.

Pre-draw the frame, not a placeholder

The skeleton mirrors the real layout the snapshot resolves into, so content settles rather than replaces. Extends the existing Skeleton primitive; no new mechanism.

Trip and Plan diverge here

The only tier that needs a mode variant: Trip’s hero is the dark board, Plan’s is the violet prep dashboard + checklist. A board skeleton that popped into a violet hero would jar — so each shape-matches. The chrome is already mode-themed at this point.

The board stays the loudest

Even as a skeleton the trip hero keeps its dark surface, so hierarchy is legible before any text loads — the eye already knows where “now/next” will be.

One announced live-region

Skeletons stay aria-hidden; the spinner label announces once. Behavior is unchanged from the shipped LoadingState — only the visual shape improves.

NN% + mini-bar (chosen)

Determinate: the same filling-line motif, shrunk into the row. An upload has a known size, so it’s honest on a slow connection abroad rather than a spinner that could mean anything. Build cost: progress wiring from the upload call (rides ADR-0056’s outbox upload).

ממתין להעלאה (offline)

No bar — nothing is uploading until the network returns. Static “waiting” is the honest state; the row stays dimmed and cloud-up until it flushes, then swaps to the determinate bar.

Replaces the מעלה… spinner

The old indeterminate spinner + word is superseded: it under-informed on flaky networks. The shared Spinner stays for the truly-unknowable cases (form submits, document open/decrypt) — just not here, where size is known.

No Trip/Plan variant here

The pending-row status is sync grammar (the --sync-* tokens + dim + cloud-up), deliberately orthogonal to the amber/teal/plan mode budget. It looks identical in both modes by design; the loaders stay neutral, never a mode hue.
