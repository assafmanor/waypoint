Trip mode · day view · proposal v1

Where are we in the day, what already happened, and does “סיימנו” still earn its tap?

A set of changes to the on-the-ground day view, all derived from the clock — nothing new is stored. A now-line anchors the current moment and splits the day into behind / ahead. The Done button is reframed from a loud hygiene verb into a quiet, optional “we did this” record. And past days read as an archive — in the list and in the top bar. Plus the boundary calls that followed review: what stays Trip-only vs. crosses into Plan, and how “new event” is scoped.

builds on ADR-0011 hard/soft ADR-0027 derived phases ADR-0028 color budget ADR-0029 past/future gating ADR-0040 finished-trip archive
01

Two clock-derived devices fix that — a live now-line, and phase treatment for what’s passed. (ADR-0027 already decided the phase model; this is where it becomes visible.)

A The now-line — dialed down

A soft amber hairline with a flat mono time label, inserted at the current moment. It stays amber (that’s the color’s meaning — time), but drops the dark chip, the glow and the pulse from v1, so it reads quieter than the live event above it.

Hierarchy is the fix for “too loud”: the amber ring on the now event stays the one accent; the line is just a reference for where the moment falls. It travels down the list as time moves — the movement is the liveness, no pulse needed.

B Passed events recede

Anything behind the line drops to a quiet, desaturated row — present for reference, no longer competing. The phase is derived from the clock, never written (deriveNow already does this for the board).

C Slipped & unresolved

A soft plan whose time passed with no tap (סנסו-ג׳י, 14:30) reads עבר · לא סומן and offers a one-tap settle right there — the honest “still on?” moment from ADR-0027, not a nag.

D The nudge adapts to phase

A small behavior the phase model makes honest: the ±30 stepper only offers moves that are actually possible.

Upcoming −30+30both ways
Now −30+30can’t pull into the past
Passed / past day −30+30retiming history is locked
Recommendation

Ship both: the now-line and the derived phase treatment. The phase model is already an accepted decision (ADR-0027) that the concurrency rewrite never surfaced — this makes it visible, and the now-line is the small addition that turns “a list of events” into “the day, and where we are in it.”

02

“סיימנו”, rethought — from board hygiene to a quiet record

Today “סיימנו” sets status = DONE: the row dims to 50%, keeps its full slot, and the board skips past it. The question is whether that tap is still worth asking for once the clock does most of the work.

What was it for?

Two jobs: hygiene (get it off the board so “now” advances) and record (we actually did this). They were bundled into one tap.

What changed?

The now-line + passed phase (§1) do the hygiene job for free — a passed event already recedes and falls out of Now/Next with no tap. Only the record job is left.

Is the record worth keeping?

Yes. Without it, a soft plan you did is indistinguishable from one you blew off — both just read “passed.” It’s the positive twin of Skip, and it’s what a future “trip wrapped” retrospective (ADR-0040) is built from.

Option A · rejected

Remove it entirely

Lean on the passed phase; never mark anything done.

  • Zero on-the-ground friction
  • “Did it” vs “skipped it” becomes unrecoverable
  • Kills the retrospective before it exists
Option B · rejected

Keep it as-is

A loud primary verb on every soft row, dims in place.

  • Familiar; no change
  • Nobody taps “done” after every meal — dead weight up front
  • Duplicates hygiene the clock now handles
Option C · recommended

Keep it, demote & reframe

An optional, affirmative “we did this” — quiet in the moment, useful behind the line.

  • Preserves the record + future retrospective
  • No friction: it’s where you already look back
  • Done settles (green ✓), not just dims

1 Ahead of now: quiet

On an upcoming or live soft event, the forward verbs lead — Navigate, Skip, nudge. סיימנו is present but last, in --ok green (a status, not amber — it isn’t a clock event). You can mark it early; you rarely need to.

2 Behind the line: where it earns its tap

On a passed-but-unmarked soft event, the settle strip surfaces the same choice as a light inline pair — ✓ היינו / דלג. One tap to make the day’s record true, exactly when you’d glance back.

3 Done reads as done

A completed event settles into a calm green-checked row instead of a fat 50%-opacity ghost holding its slot. Reversible via the row’s שחזר, unchanged.

Recommendation

Keep סיימנו — demoted and reframed (Option C). Never auto-write it (ADR-0027 holds). Its natural home is the behind-now settle zone and the past-day archive (§3), where “did we?” is a real question and one tap answers it.

D An archive wash, not a dead screen

The whole list cools and desaturates slightly, led by a לקריאה בלבד banner. No now-line (there is no “now” on a day that’s over), no amber. It reads as memory the instant it loads.

E Editing is locked, settling isn’t

Per ADR-0029, structural edits are gated to Plan mode (the dashed footer says so). But Done / Skip / Navigate stay — an unresolved soft plan can still be settled with one tap, which is exactly the retrospective job the reframed Done was built for.

F Amber marks today, not the selection

The systemic fix in the top bar: amber is the trip’s live anchor, so it stays on today’s pill wherever you browse. Selecting a past day is a neutral highlight; a future day is violet (plan-ahead). One glance always finds “now,” and history never borrows the live color (ADR-0028).

Recommendation

Reuse this same archive treatment for a finished trip (ADR-0040) — it’s the whole trip in read-only, one day at a time — so past-day and past-trip share one visual language instead of two.

While we’re in the top bar — the account & settings cluster

A small tidy-up you flagged: today the settings gear is a third filled circle sitting beside the identity avatars, so three round chips compete. Separating identity from controls reads cleaner.

04

Where the line is drawn — Plan mode, and the “new event” button

Two boundary questions, one rule behind both: Trip mode is live and on-the-ground; Plan mode is the drafting table and is never “live.” (design-language: mode identity.)

Do these features cross into Plan mode?

Mostly no — and that’s a feature, not a gap. Only the archive language crosses.

G A reference, not a heartbeat

Same information (“here’s now”), opposite grammar. Trip’s now-line is a solid amber hairline with a filled dot — live. Plan’s is a dashed violet line with a hollow ring and no pulse — a drafting guide. You can’t confuse the drafting table for the board.

H Only where it helps

It appears only when the day on screen is today and the trip is live — the one case where “now” means something while building. Past and future builder days show no reference (there’s no now to place), keeping ADR-0029’s “Plan has no past/future distinction” intact for editing.

Recommendation

Add it, in Plan’s violet. It earns its place by helping you build around the clock without borrowing the live board’s amber — mode identity holds.

Should Trip mode keep “אירוע חדש”?

Don’t remove it — right-size it. The need is real (“we just decided to grab coffee at 5”), but today’s button opens the full builder.

✓ Keep it, scoped to Tier-1

ADR-0025 lists “add a quick soft event for today” as a legitimate on-the-ground verb. So keep an add affordance — but make it a quick-add (soft, today, next open slot), not the full form. Adding a hard event, a booking, or an event on another day is building → the ⋯ sheet or the Plan escape.

✕ Gone on past days

On a past day ADR-0029 locks create — so the add button disappears entirely, replaced by §3’s “add or move in Plan mode” footer. The archive stays read-only.

Recommendation

Keep it on today, right-sized; hide it on past days. Net effect: spontaneous soft adds stay one tap on the ground, real itinerary building lives where it belongs — Plan mode.
