מה נתמך — ומה נבנה

העיצוב סוכם: מנהלים שולטים בטיול. חלק מהפעולות כבר עם backend חי, וחלק דורשות עבודת backend חדשה (מסומן). מה שעדיין לא מוכר מסומן כעתידי.
נתמך היום נבנה · צריך backend עתידי
✓ נתמך היום
רשימת חבורה + תפקידים · הזמנה (POST /trips/:id/invite) · יציאה מהטיול והסרת משתתף ע״י מנהל (DELETE /trips/:id/members/:userId, כבר admin-gated ב-service).
🔄 כל שינוי — realtime + offline
כל מוטציה (פרטים / תפקיד / הסרה / מחיקה) עוברת דרך ChangeService.mutate: כתיבה + רשומת Change בטרנזקציה אחת, ואז broadcast ב-WS אחרי commit — מופיע מיד לכולם. בקליינט: אותו דפוס אופטימי כמו verbs.ts (dispatch → REST → reconcile/rollback) + outbox, כך שעובד offline ומסתנכרן אחר-כך.
פערים לסגור: ה-membership ops היום עוקפים את ChangeService (prisma ישיר, בלי broadcast); והקליינט מיישם רק entityType==='event' (trip-state / cache) + ל-outbox יש רק verbs של אירועים. צריך 'trip' ו-'membership'.
⚒ עריכת פרטים — מנהל בלבד
שם / יעד / תאריכים כטופס אחד → PATCH /trips/:id חדש, admin-gated בשרת (לא רק ב-UI). כפתור "עריכה" מוסתר למשתתף. שינוי טווח תאריכים → אזהרה על אירועים שיוצאים מהטווח.
📅 שדות תאריך — לאמץ מ-PR #92
אותו תיקון date-picker שמוזג ב-CreateTrip/EventForm: lang=DEVICE_LOCALE (פורמט מקומי ולא mm/dd/yyyy כמו שרואים כאן במוקאפ), min של הסיום קשור להתחלה, ו-refine של endDate ≥ startDate. בלי רצפה להיום — עריכת טיול קיים חייבת לאפשר תאריכים שכבר עברו/התחילו.
⚒ קידום למנהל 👑
מנהל מקדם משתתף למנהל. צריך PATCH /trips/:id/members/:userId (role), admin-gated. אין הורדת-דרגה מפורשת ב-v1 בכוונה.
⚒ מחיקת טיול — מנהל בלבד
DELETE /trips/:id חדש, admin-gated, אישור כפול. מוחק לכולם.
⚒ מנהל אחרון עוזב
כשמנהל אחרון יוצא — משתתף אחר מקודם אוטומטית (שרירותי) בתוך removeMember. כך אין טיול "יתום" בלי מנהל.
אזור-זמן + מטבע — נערכים ידנית (עכשיו)
נכללים בטופס העריכה כרגע (select), עד שייגזרו אוטומטית מהיעד בעתיד. אז צריך לחשוב על המנגנון (Google Places? מיפוי יעד→TZ/מטבע) והאם להשאיר override ידני.
Calendar sync / Gmail / Photos / WhatsApp
Calendar: calendarSyncEnabled נשמר אבל ה-push לא מחווט (ADR-0003). Gmail = v1.1; Photos/WhatsApp מחוץ לסקופ (אינטגרציות הן צינורות, ADR-0004). כולם מושמטים כדי לא להציג פקד שלא עובד.
טוגלים של התנהגות (ripple, זיהוי-קשיח)
התנהגויות מובנות, לא העדפות נשמרות — אין להן שדה. שליטה תדרוש מודל הגדרות-טיול חדש.
Chrome: נייטרלי (סוכם)
כותרת נייטרלית ink-on-paper — הגדרות נגישות משני המצבים. לא נושא זהות-מצב.
