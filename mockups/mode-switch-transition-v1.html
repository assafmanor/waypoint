Plan ⇄ Trip: the lights change

The mode switch is the app's most meaningful moment — Day (the drafting table) becoming Night (the departure board powering on), and back. Today only the board's power-on exists; the chrome swap is abrupt and the return trip is undesigned. Below: five ways to animate that lighting change on a live phone, a recommendation, and the smaller transitions that should share its vocabulary.

Direction matters: Plan→Trip is the arrival — the celebrated beat. Trip→Plan is a quiet return to the desk, so it runs faster.

The rest of the motion system

The mode switch is the one cinematic moment — everything else stays quick and quiet, the motion analogue of "one loud element." These smaller transitions should all draw from the same duration + easing tokens so the app feels like one hand made it. Tap each to preview.

Proposed motion tokens

A small ramp of durations and easings — the shared vocabulary. New motion picks from these instead of inventing values, the same discipline the type ramp and color budget already enforce.

TokenValueUsed for
--t-quick140msNav settle, toggles, hovers, focus rings — the "instant but not jarring" tier.
--t-base240msTab cross-fade, toast, ripple bar, sheets, most enters/exits.
--t-deliberate400msReturn-gesture screen slide (ADR-0035), and Trip→Plan (the quiet return).
--t-cinematic600msPlan→Trip — the board wakes. The only cinematic transition in the app.
--ease-standardcubic-bezier(.2,0,0,1)Default. Entrances, color melts, most everything.
--ease-exitcubic-bezier(.4,0,1,1)Things leaving — toast out, dismissals.
--ease-emphasizedcubic-bezier(.16,1,.3,1)The board wake — a confident settle with a soft landing.
The budget rule: exactly one --t-cinematic moment exists in the product — the Plan→Trip switch. Spending "cinematic" anywhere else devalues it everywhere, just like amber or teal.
Recommendation

Ship B — "Board wakes", direction-aware

It's the only candidate that tells the product's story: the paper dims, the board powers on (the seed already in screens.css), the clock starts, amber warms in last, the pulse begins. Asymmetric by design — arrival is celebrated, the return is calm.

  • BPlan→Trip runs the two-beat wake at 600ms: chrome→indigo, then board brightness+glow rise, accents + pulse arrive on the tail.
  • BTrip→Plan is the quiet inverse at 400ms: glow fades, board flattens to paper, the drafting grid resolves. No fanfare — you're back at the desk.
  • EUse Temperature shift for the automatic date-driven switch (no tap to celebrate) — a silent 280ms warm/cool so a mode flip you didn't ask for never startles.
  • ACrossfade is the graceful floor and the reduced-motion answer collapses to it — an instant token remap, mode identity intact.
  • ✕Skip D (Ripple) — the expanding circle reads Material/consumer-flashy against a calm travel companion. C (Sweep) is lovely but optional polish over B, not a base.

Grounded in design-language.md (Night/Day mode identity, "the plan→trip switch is the product's most meaningful moment"), ADR-0016 (derived + per-user override switch), and ADR-0028 (the plan-violet budget). This is a proposal — adoption means a "Motion & designed transitions" update to the design language plus the CSS wiring (transition vars on the .app chrome + a direction-aware class on the mode flip). Fonts here are a system substitute for Secular One / Assistant / JetBrains Mono; motion, not type, is the subject. — Assaf's Waypoint, v1.
