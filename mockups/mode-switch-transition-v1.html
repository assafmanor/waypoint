Plan ⇄ Trip: temperature & energy, not lights

The mode switch is the app's most meaningful moment — but it is not a light-to-dark flip. Mode rides on channels that survive a theme change: the chrome's violet cooling to indigo + amber, the drafting grid dissolving, and the board's glow & pulse going live. Luminance belongs to a separate axis — the light/dark theme, future work — where both modes go dark (plan = violet-tinted dark, trip = indigo dark). So every candidate below must read identically in either theme. Flip the theme in the console to prove it.

Mode axis — carries identity
Temperature violet ⇄ indigo+amber
Drafting grid plan only
Glow + pulse trip "live"
Theme axis — orthogonal, future: light ⇄ dark. The transition never touches it.

Proposed motion tokens

A small ramp of durations and easings — the shared vocabulary. New motion picks from these instead of inventing values, the same discipline the type ramp and color budget already enforce.

Token Value Used for
--t-quick 140ms Nav settle, toggles, hovers, focus.
--t-base 240ms Tab cross-fade, toast, ripple bar, sheets.
--t-deliberate 400ms Return-gesture slide (ADR-0035); Trip→Plan stand-down.
--t-cinematic 600ms Plan→Trip going live. The only cinematic transition.
--ease-standard cubic-bezier(.2,0,0,1) Default / entrances / hue melts.
--ease-exit cubic-bezier(.4,0,1,1) Exits — toast out, glow extinguishing.
--ease-emphasized cubic-bezier(.16,1,.3,1) The glow ignite — a confident settle.
The budget rule: exactly one --t-cinematic moment exists — the Plan→Trip switch. Spending "cinematic" elsewhere devalues it, same discipline as amber / teal / violet.
Recommendation

Ship B — "Go live / Stand down", direction-aware & theme-independent

It choreographs the durable channels — temperature, grid, and the board's glow/pulse — and touches luminance nowhere, so it reads the same in light or dark theme. The body stays paper; only the chrome hue, the hero, the grid, and the glow move.

  • BPlan→Trip (going live), 600ms: chrome warms violet→indigo and the grid dissolves, then the board's amber glow ignites and the pulse starts — the climax lands on the "live" energy, not on brightness.
  • BTrip→Plan (stand-down), 400ms: the glow extinguishes and the pulse stops first, then indigo cools to violet and the drafting grid draws itself back. Quiet — you're back at the desk.
  • ✓Survives the theme. In dark mode both surfaces are dark; the same hue-shift + grid + glow choreography still separates plan from trip. Toggle the theme mid-switch to confirm.
  • ACrossfade is the graceful floor and the reduced-motion answer collapses to it — an instant token remap, mode identity intact.
  • ↺Use a gentler, non-staged version (closer to A, ~280ms) for the automatic date-driven switch — a mode flip the user didn't ask for shouldn't perform.
  • ✕An expanding-circle "ripple from the toggle" was tried and cut — reads consumer-flashy, and there's no tap to originate from on the auto switch.

Grounded in design-language.md (Night/Day mode identity + Dark-mode readiness: "trip stays indigo, plan is violet-tinted dark"), ADR-0016 (derived + per-user override switch), and ADR-0028 (the plan-violet budget).
